# Security Policy

## Supported Versions

Only the latest code on the default branch is considered supported.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security-sensitive problems.

Report details privately to the repository maintainer with:

- a clear description of the issue
- impact and affected endpoints
- reproduction steps or a proof of concept
- any suggested mitigation

This project is designed for trusted local environments by default. If you
deploy it beyond localhost, you are responsible for adding authentication,
network controls, and any required transport security.
